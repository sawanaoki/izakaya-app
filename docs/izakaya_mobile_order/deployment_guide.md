# 公開・デプロイ方法ガイド

「みんなに見てもらう」ためには、インターネット上からアクセスできるようにする必要があります。
大きく分けて、**「一時的に自分のPCを公開する方法（Ngrok）」**と**「クラウドサーバーに設置する方法（Vercel）」**の2つがあります。

---

## 方法1: Ngrok（エングロック）を使って一時公開
**おすすめ**: 今すぐ友人やクライアントに見せたい場合。
自分のPCで動いているアプリを、インターネット経由で見れるようにするトンネルを作ります。

### 手順
1. **Ngrokのインストール**:
   - 公式サイト [ngrok.com](https://ngrok.com/) でアカウント作成（無料）。
   - 手順に従ってインストールとコマンド設定（`authtoken`）を行います。

2. **アプリの起動**:
   - 別のターミナルで `npm run dev` を実行しておきます。

3. **公開**:
   - 新しいターミナルで以下を実行します：
     ```bash
     ngrok http 3000
     ```
   - 表示された `Forwarding` の横にあるURL（例: `https://xxxx-xxxx.ngrok-free.app`）を相手に教えます。

### メリット・デメリット
- ✅ **メリット**: コードやデータベース（SQLite）の変更が不要。すぐにできる。
- ❌ **デメリット**: 自分のPCを起動しておく必要がある。無料版は起動ごとにURLが変わる。

## 方法2: Vercelに正式デプロイ
**おすすめ**: ポートフォリオとして常に公開したい場合。
Next.jsの開発元であるVercelを使うのが標準的ですが、**データベースの変更が必要です**。

### 注意点: データベースについて
現在の **SQLite** は、Vercelのようなサーバーレス環境ではデータが保存されません（再起動するたびにデータが消えます）。
そのため、**PostgreSQL**（Vercel Postgres や Neon など）に変更する必要があります。

### 手順の概要
1. **データベースの切り替え**:
   - Vercel Postgresなどのクラウドデータベースを用意。
   - `prisma.schema` を `postgresql` に書き換え。
   - マイグレーションを実行。
2. **Vercelと連携**:
   - VercelにGitHubリポジトリをインポート。
   - 環境変数を設定してデプロイ。
