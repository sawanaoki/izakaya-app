# 実装計画：居酒屋モバイルオーダーシステム

## 目的
居酒屋向けの注文管理システムを開発します。管理者がメニューを登録・QRコードを発行し、客がスマートフォンからQRコードを読み取って注文できるWebアプリケーションを作成します。

## ユーザーレビューが必要な項目
- **データベース**: ローカル動作を前提とし、SQLiteを使用します（セットアップ不要で便利です）。
- **ネットワーク**: 実際にスマホから注文するには、スマホとPCが同じWi-Fiネットワークにあり、ファイアウォール設定が必要な場合がありますが、今回は「開発・動作確認」を主眼に置き、localhostでの動作を基本とします。

## 変更内容

### 技術スタック
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Styling**: Tailwind CSS (リッチなデザインに使用)
- **Database**: SQLite (Prisma ORM)
- **Icons**: Lucide React

### コンポーネント構成

#### 1. データベース (`prisma/schema.prisma`)
- `Category`: メニューカテゴリ
- `MenuItem`: 商品（名前、価格、画像、カテゴリ、在庫状況）
- `Order`: 注文履歴（テーブル番号、ステータス）
- `OrderItem`: 注文詳細

#### 2. 管理画面 (`/admin`)
- **ダッシュボード**: 現在の注文状況をリアルタイム（ポーリング）表示
- **メニュー管理**: 商品の追加・編集・削除、画像のアップロード（ローカル保存）
- **QRコード生成**: テーブルごとの注文用QRコード表示・印刷

#### 3. 客用画面 (`/order/[tableId]`)
- **メニュー閲覧**: カテゴリごとのきれいな表示
- **カート機能**: 注文内容の確認
- **注文送信**: 確定してサーバーへ送信

## 検証計画

### 自動テスト
- 現段階では基本機能の実装を優先し、Unit Testの実装は最小限とします。

### 手動検証
1. **管理者機能**:
   - メニュー登録ができるか
   - QRコードが表示されるか
2. **注文フロー**:
   - QRコード（URL）にアクセスし、メニューが表示されるか
   - カートに入れて注文できるか
   - 管理画面に注文が反映されるか
