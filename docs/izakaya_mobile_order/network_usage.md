# ネットワーク環境での使用方法

このアプリケーションをスマホなど他の端末から利用するには、以下の手順が必要です。

## 1. 同じWi-Fiに接続する
PCとスマホが同じWi-Fi（ローカルネットワーク）に接続されていることを確認してください。

## 2. IPアドレスの確認
PCのIPアドレスを確認します。
PowerShellで以下のコマンドを実行してください：
```powershell
ipconfig
```
`IPv4 アドレス . . . . . . . . . . : 192.168.x.x` のような行を探します。

## 3. アプリケーションの起動
Next.jsをネットワーク上の他のデバイスからアクセス可能にするため、ホストを指定して起動します。

`package.json` の `scripts` に以下を追加するか、直接コマンドを実行します：
```bash
npm run dev -- -H 0.0.0.0
```
※ `-H 0.0.0.0` は「すべてのネットワークインターフェースからの接続を受け付ける」という意味です。

## 4. ファイアウォールの許可
初回起動時、Windowsファイアウォールの警告が出た場合は「アクセスを許可する」を選択してください。
もし繋がらない場合は、一時的にファイアウォールを無効にするか、ポート3000の受信を許可する必要があります。

## 5. スマホからのアクセス
スマホのブラウザで `http://[PCのIPアドレス]:3000/order/1` にアクセスします。
例: `http://192.168.0.10:3000/order/1`

## 便利な設定（推奨）
現在、QRコードの発行URLは `localhost` 固定になっています。これを実際のIPアドレスに自動追従させるには、環境変数や設定変更が必要です。
今回は簡易的に、`.env` ファイルに `NEXT_PUBLIC_BASE_URL` を設定することをお勧めします。

1. `.env` ファイルに以下を追記（自分のIPアドレスに書き換えてください）:
```
NEXT_PUBLIC_BASE_URL=http://192.168.0.x:3000
```

2. `src/app/admin/qr/page.tsx` を修正して、この環境変数を使用するようにします。
